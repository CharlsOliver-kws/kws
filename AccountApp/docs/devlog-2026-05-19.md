# 2026-05-19 开发记录

## 一、图片识别模型与功能

### 新增能力
- 接入阿里云 DashScope Vision Language 模型（兼容 Anthropic API 端点）
- 用户拍照/选择图片 → VL 模型识别 → 返回多条账单 JSON → 确认弹窗编辑 → 逐条入 SQLite

### 技术实现
- **`ImageBillService`** (`lib/core/services/image_bill_service.dart`)
  - 使用 `image_picker` 获取图片，base64 编码后 POST 到 VL API
  - Prompt 设计：要求模型返回 JSON 数组，每条含 amount/category/note/date
  - 内置重试机制（最多 2 次），容错空日期/空字段
  - 兼容 Anthropic 协议端点 `dashscope.aliyuncs.com/apps/anthropic`
- **`ImageBillDialog`** (`lib/features/home/widgets/image_bill_dialog.dart`)
  - 识别结果列表展示，支持单条编辑（底部弹窗）、单条删除、全部确认添加
  - 编辑面板：金额、分类（ChoiceChip）、日期、时间、备注均可改
- **`DateHelper.parseDateTime`** (`lib/core/utils/date_helper.dart`)
  - 解析 VL 返回的多种日期格式（`YYYY-MM-DD HH:mm` / `YYYY-MM-DD`）

### 数据流
```
用户拍照 → ImageBillService.recognizeBills(path)
  → VL API (base64 图片 + prompt)
  → 解析 JSON → List<Record>
  → ImageBillDialog 确认/编辑
  → RecordsNotifier.addRecord(record) × N
  → SQLite insert + Widget 刷新 + LockScreen 刷新
```

### 遇到的技术问题

#### 1. Open Design 频繁崩溃（exit code 1）
- **现象**：生成 Dart 代码时连续 crash 3+ 次，尤其是大文件
- **原因**：
  - 未安装 pnpm（Open Design 内部用 pnpm 管理）
  - AI 代理在处理复杂 Flutter 代码时不稳定
- **解决**：
  - 安装 `pnpm@10.33` + 配置 `PNPM_HOME` 环境变量
  - 多次重试，崩溃后继续生成

#### 2. 模型不匹配：Open Design Record vs 现有 Record
- **Open Design 的 Record**：
  - `String id` (UUID 风格)
  - `CategoryId category` (enum)
  - `DateTime time` (字段名是 time)
  - 无 voiceText/voiceFile/createdAt/updatedAt
- **现有 Record** (`lib/data/models/record.dart`)：
  - `int? id` (SQLite auto-increment)
  - `String category` (中文字符串 "餐饮"/"交通"/...)
  - `DateTime date` (字段名是 date)
  - 有 voiceText/voiceFile/createdAt/updatedAt
- **当前状态**：需要统一，后端用现有 Record（SQLite 兼容），前端组件用现有 Record 传数据

#### 3. 分类系统两套并存
- **Open Design**：`CategoryId` enum（food/transport/shopping/...），颜色和背景色在 `AppColors` 中，有 `catColor()` / `catBg()` / `fromId()` 方法
- **现有代码**：`Categories` 常量类（'餐饮'/'交通'/'购物'/...），`iconFor()` / `colorFor()` 静态方法
- **解决方向**：保留现有 `Categories` 类作为数据层使用（数据库存中文），UI 层直接用 Open Design 的 color/icon 映射

#### 4. image_confirm_screen.dart 的 Dummy 类型 bug
- `_DummyDate` 和 `_DummyTime` 类未正确实现 `DateTime` / `TimeOfDay` 接口，会导致编译错误
- **需要修复**：改为 `DateTime.now()` / `TimeOfDay.now()` 默认值

#### 5. home_screen.dart 内有重复的 Record/RecordGroup 定义
- Open Design 的 `home_screen.dart` 自己定义了 Record/RecordGroup（不同于 models.dart 中的版本）
- **需要处理**：需要替换为引用现有 `lib/data/models/record.dart`

#### 6. image_confirm_screen.dart 使用 Map<String, dynamic> 而非 Record 类
- Open Design 版本用 `Map<String, dynamic>` 传数据，缺少类型安全
- **现有版本更好**：`ImageBillDialog` 已经使用 `List<Record>`，编辑时直接操作 Record 对象

## 六、Open Design 集成踩坑全记录

### 阶段一：HTML/CSS 方案失败

最初尝试将 Open Design 生成的 HTML/CSS 原型直接引入 Flutter：
- 使用 `flutter_html` / `webview_flutter` 渲染 HTML
- **失败原因**：HTML 布局在 Flutter 中严重失真，CSS 样式大量不支持，交互事件无法传递。结论：HTML/CSS 方案不可行。

### 阶段二：转为 Dart Widget 方案

要求 Open Design 直接生成 Flutter Dart 代码（`appfront/dart/flutter_widgets/`），共 11 个文件。接入过程遇到以下问题：

#### 1. 编译错误
| 错误 | 原因 | 修复 |
|---|---|---|
| `CardTheme` / `DialogTheme` 未定义 | Flutter 已改名为 `CardThemeData` / `DialogThemeData` | 全局替换 |
| `Color` 未定义 | `design_tokens.dart` 缺少 `import 'package:flutter/material.dart'` | 补充 import |
| `_DummyDate` 实现 `DateTime` 失败 | `DateTime` 有大量抽象方法无法 stub | 改为 `DateTime?` 空值 |

#### 2. 布局溢出
| 位置 | 溢出量 | 修复 |
|---|---|---|
| `settings_screen.dart` SegmentedButton | 116px | Row → Column，按钮独占一行 |
| `image_bill_dialog.dart` 账单行 | 严重 | 单 Row → 嵌套双 Row（金额/操作分两行） |
| `image_bill_dialog.dart` actions | 14px | Row → Column（合计在上，按钮在下） |

#### 3. 功能缺失
- **语音按钮无响应**：`_BottomNav` 中 `onTap: () {}` 为空 → 替换为 `VoiceButton` 组件，用 `Listener` 监听 pointer down/up 控制录音
- **图片识别乱码**：已修复（`ImageBillDialog` 中正确显示金额和分类）

#### 4. Android Native 层问题

**锁屏通知连续崩溃（3 次迭代）**：
1. `BadNotificationException: View tag not supported` → 移除所有 `android:tag`
2. `BadNotificationException: Space tag not supported` → 移除 `<Space>` 标签
3. `BadNotificationException: alpha attribute` → 移除 `android:alpha`

最终 `notif_lockscreen.xml` 只保留 `LinearLayout` + `TextView`，所有属性均为 RemoteViews 兼容子集。

**Kotlin 字符串插值错误**：
- `$todayCount笔` → Kotlin 将 `todayCount笔` 视为变量名 → 改为 `${todayCount}笔`

**桌面小组件崩溃**：
- `View` 标签不支持 → 用 `FrameLayout` 替代
- 文字换行（28sp 太大）→ 降至 18sp + `singleLine="true"`
- 分隔线 `View` 不支持 → 用 `FrameLayout`（1dp 宽 + 背景色）

#### 5. 视觉对齐修复
- **金额颜色**：Open Design 要求金额用 `#1A1E2B`（fg 色），不是绿色 → `home_screen.dart` / `all_records_screen.dart` 统一 `cat.color` → `AppColors.fg`
- **备注显示**：Open Design 直接显示 `record.note`，不 fallback 到分类名 → 统一为 `record.note`

### 阶段三：最终架构决策

- **数据层**：保留现有 `lib/data/models/record.dart`（SQLite 兼容，中文分类名）
- **UI 层**：用 Open Design 的 `design_tokens.dart`（颜色/圆角/间距/字体）统一视觉
- **组件层**：现有功能组件（`ImageBillDialog`、`ManualEntryDialog`、`VoiceButton`）保留，UI 风格向 design tokens 对齐
- **不直接替换** `home_screen.dart` 为 Open Design 版本（模型不匹配成本太高），而是逐组件吸收设计元素

### Open Design 踩坑总结

今天重点回顾并记录了 Open Design 集成的三个阶段的完整踩坑过程：

| 阶段 | 方案 | 结果 |
|---|---|---|
| 一 | HTML/CSS 直接引入 Flutter | 失败 — 布局失真、CSS 不兼容、事件不通 |
| 二 | Open Design 生成 Dart Widget（11 个文件） | 部分成功 — 需大量修编译错误/布局溢出/功能缺失 |
| 三 | 逐组件吸收设计元素 | 当前方案 — 数据层保留现有，UI 对齐 design tokens |

### 关键教训

1. **RemoteViews 限制严格**：Android 锁屏通知和桌面小组件只能用 `LinearLayout` + `TextView` 子集，连 `<View>`、`<Space>`、`alpha` 都不支持，调试全靠崩溃日志逐条排查
2. **Kotlin 字符串插值陷阱**：`$variable中文` 会被解析为 `$variable中文`（整体当变量名），必须加花括号 `${variable}中文`
3. **Open Design 模型不匹配**：AI 生成代码的数据模型与现有 SQLite 模型字段名/类型不一致，直接替换成本太高，逐组件吸收更实际
4. **布局溢出是高频问题**：手机屏幕窄（约 360dp），多个组件挤在 Row 里必溢出，需要拆成 Column 或嵌套 Row
