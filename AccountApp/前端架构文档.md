# 语音记账 - 组件与交互逻辑

## 一、页面路由

| 路由 | 页面 | 入口方式 |
|------|------|----------|
| `/` | 首页 | 应用启动默认进入 |
| `/settings` | 设置页 | 首页底部"设置"按钮 |
| `/all-records` | 全部记录 | 首页底部"记录"按钮 |

返回通过 AppBar 返回按钮或系统手势。

---

## 二、首页

### 整体结构

```
AppBar
  └── 左侧: 应用图标 + 应用名
  └── 右侧: 手动记账入口按钮(icon)

内容区 (可滚动区域)
  └── 汇总卡片
  └── 记录列表（最多6条，按天分组）

底部固定按钮栏 (始终悬浮在内容上方)
  └── 按钮1: "记录" → 跳转全部记录页
  └── 按钮2: "语音" → 长按录音（核心交互）
  └── 按钮3: "识图" → 拍照/相册识别账单
  └── 按钮4: "设置" → 跳转设置页

全屏遮罩层 (条件显示)
  └── 录音时覆盖整个屏幕
```

### 组件说明

#### 汇总卡片
- 展示"今日支出"和"本月累计"两个数据
- 数据来源于本地数据库中的 Record 列表，实时计算

#### 记录列表
- 数据源：数据库中所有 Record，按日期倒序
- 每条记录显示：分类图标、备注/分类名、时间、金额
- **分组规则**：同一天归为一组；"今天"/"昨天"显示为相对日期，其余显示"X月X日 星期X"
- **截断规则**：首页最多展示 6 条，超出部分底部显示"查看更多"入口
- 列表底部预留空间，避免被底部按钮遮挡

#### 记录条目交互
- **左滑删除**：左滑露出删除按钮 → 点击后弹出确认弹窗 → 确认后删除并刷新列表
- **点击**：当前无操作（预留）

#### 空状态
- 无记录时显示引导图标 + "暂无记录" + "点击底部按钮开始记账"

## 三、语音记账交互

### 交互流程

```
用户长按"语音"按钮
  → 按钮进入按下态（视觉反馈）
  → 开始录音
  → 全屏录音遮罩出现：
      - 麦克风大图标
      - "正在录音..." 文字
      - 动态波形动画
      - "松开完成录音" 提示
  → 用户松开
      → 录音结束，遮罩消失
      → 显示 Loading 转圈
      → 音频文件发送到阿里云 ASR 转文字
      → 文字发送到 AI 模型解析为 {金额, 分类, 备注}
      → 解析结果写入数据库
      → 刷新记录列表
      → SnackBar 提示"记录已添加"

特殊情况:
  - 用户滑动取消（手指移出按钮区域）→ 取消录音，不保存
  - 解析失败 → SnackBar 显示错误信息
```

### 涉及组件
- **VoiceButton**：使用 Listener 捕获指针按下/抬起/取消事件
- **RecordingOverlay**：全屏遮罩，在 Stack 中根据录音状态条件渲染

## 四、手动记账

### 入口
- 首页 AppBar 右上角 "+" 图标按钮

### 交互流程

```
点击"+"按钮
  → 弹出 Dialog 表单：
      1. 金额输入框（数字键盘）
      2. 分类选择区（网格排列，8个分类，点击选中/切换）
      3. 日期选择器
      4. 时间选择器
      5. 备注输入框（选填）
      6. "保存"按钮
  → 点击保存
      → 校验：金额不能为空且必须 > 0
      → 通过：写入数据库 → 关闭弹窗 → 刷新列表 → SnackBar"记录已添加"
      → 失败：SnackBar 提示具体错误
```

### 分类选择交互
- 网格布局展示 8 个分类（餐饮/交通/购物/娱乐/医疗/教育/住房/其他）
- 每个分类显示图标 + 名称
- 点击某个分类即选中，再次点击其他分类切换
- 选中态与未选中态有视觉区分

## 五、图片识别记账

### 入口
- 首页底部"识图"按钮

### 交互流程

```
点击"识图"按钮
  → 弹出选择弹窗：
      - "从相册选择" → 调起系统相册
      - "拍照" → 调起系统相机
      - "取消"
  → 用户选图/拍照后
      → 显示 Loading 转圈
      → 图片转为 Base64 发送到视觉大模型
      → 模型返回 JSON 数组（金额、分类、备注、日期）
      → 解析为 Record 列表
  → 弹出确认弹窗：
      - 标题："识别到 N 笔账单"
      - 逐条展示识别结果
      - 每条可编辑（点击弹出编辑面板）
      - 每条可删除（点击 × 按钮）
      - 底部：合计金额 + "确认添加"按钮
  → 确认添加
      → 逐条写入数据库
      → SnackBar"已添加 N 条记录"
  → 识别失败
      → SnackBar 显示错误
  → 识别到0条
      → SnackBar"未能识别到账单信息，请重试"
```

### 确认弹窗中的编辑交互

点击某条记录 → 弹出底部编辑面板：
- 金额输入
- 分类选择（标签式排列）
- 日期选择
- 时间选择
- 备注输入
- "保存"按钮 → 更新该条数据，关闭面板

### 重试机制
- API 调用失败自动重试 1 次（共 2 次）

## 九、数据流

```
用户操作（录音/手动输入/图片识别）
  → AI 服务解析（语音转文字 → 文本解析 / 视觉识别）
  → Record 对象
  → RecordRepository.insert()
  → SQLite 数据库
  → RecordsNotifier.loadRecords() 刷新列表
  → UI 自动更新（Riverpod AsyncValue）
  → 同时触发：WidgetUpdateService（桌面小组件）
  → 同时触发：LockScreenService（锁屏通知）
```

### Provider 依赖关系

```
voiceServiceProvider       → 提供录音和转写能力
voiceStateProvider         → 管理录音状态 (idle/recording) 和文件路径
recordRepositoryProvider   → 提供数据库 CRUD
recordsProvider            → 管理记录列表（AsyncValue），驱动 UI 刷新
themeModeProvider          → 管理主题模式
```

## 十、桌面小组件 (Home Screen Widget)

### 入口
- 用户长按手机桌面空白处 → 选择"小组件" → 找到"语音记账"拖拽到桌面
- 首次启动 App 时会弹出引导说明

### 组件结构

```
小组件卡片 (固定尺寸，由系统网格决定)
  ├── 左侧: 今日支出金额 + "今日支出"标签
  ├── 右侧: 本月累计金额 + "本月累计"标签
  └── 底部: 最近一笔记录的备注/分类名（可选）
```

### 数据更新机制
- 每次添加/删除记录后，App 调用 HomeWidget API 更新小组件数据
- 数据字段：`today_amount`、`today_count`、`monthly_amount`、`monthly_count`
- 小组件点击 → 打开 App 首页

### 涉及技术
- `home_widget` Flutter 插件
- Android: `AccountWidgetProvider` (Kotlin)
- iOS: App Group + WidgetKit (Swift)
- 数据通过 `HomeWidget.saveWidgetData()` 写入，系统定时刷新

## 十一、锁屏通知 / 下拉通知栏

### 入口
- 用户下拉系统通知栏可见
- 在设置页开启"锁屏通知"开关后生效

### 组件结构

```
通知卡片 (系统通知样式)
  ├── 应用图标 + "语音记账" 标题
  ├── 今日摘要:
  │     ├── "今日支出: ¥XX.XX (N笔)"
  │     └── "本月累计: ¥XX.XX (N笔)"
  └── Action: 点击 → 打开 App 首页
```

### 显示/更新机制
- 每次添加/删除记录后自动刷新通知内容
- 设置页关闭开关 → 立即隐藏通知（`LockScreenService.hide()`）
- 设置页开启开关 → 立即显示通知（`LockScreenService.show()`）
- 用户清除通知后，下次记录变更时重新显示

### 交互
- **点击通知** → 打开 App 首页
- **滑动清除** → 移除通知（可下次记账时重新出现）
- **长按通知** → 进入系统通知设置（关闭/优先级/静默等）

### 涉及技术
- Flutter `MethodChannel` → Android NotificationManager / iOS UNUserNotification
- Android: 前台服务常驻通知（Foreground Service Notification）
- iOS: UNNotification 推送，需用户授权通知权限
