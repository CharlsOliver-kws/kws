## 4. 连接步骤

### 方式一：配对连接（首次使用）

1. 打开手机 **设置 → 开发者选项 → 无线调试 → 使用配对码配对设备**
2. 记下手机显示的 **IP 地址** 和 **配对码**
   - 手机 IP：`192.168.1.xxx`
   - 配对码：如 `452737`
3. 在电脑执行：

```powershell
# 进入 ADB 路径
cd "C:\Users\kws\AppData\Local\Android\sdk\platform-tools"

# 配对（交互模式下输入配对码）
.\adb.exe pair <手机IP>:<端口>
# 输入配对码 452737

# 配对完成后连接
.\adb.exe connect <手机IP>:<端口>
```

> **注意**：adb pair 是交互式命令，需要在终端中手动输入配对码，不能通过管道传入。

### 方式二：无需配对码连接（已配对过的设备）

如果手机之前已配对过，直接连接即可：

```powershell
.\adb.exe connect 192.168.1.102:42579
```

### 方式三：通过配对码设备名连接

如果 adb 已通过 mDNS 发现设备：

```powershell
.\adb.exe connect adb-c367363e-9IoM3S._adb-tls-connect._tcp
```

## 7. 常见问题

### 手机息屏后无线连接断开

一加手机息屏后 adb 无线连接会断开，需要重新配对连接。解决方法：

```powershell
.\adb.exe connect <手机IP>:<端口>
```

如果连接失败，重新走配对流程。

### adb pair 提示 No error

`adb pair` 是交互式命令，不能在脚本中通过管道自动输入配对码。需要：

1. 直接在 **CMD / PowerShell 终端** 手动运行 `adb pair`
2. 等待出现 `Enter pairing code:` 提示后手动输入 `452737`

### flutter doctor 显示设备离线

```powershell
# 重启 adb server
.\adb.exe kill-server
.\adb.exe start-server

# 重新连接
.\adb.exe connect <手机IP>:<端口>
```

### Android 许可证问题

```powershell
flutter doctor --android-licenses
# 按 y 接受所有许可
```
